# S10 — Cut and publish 1.3.0 (runs LAST, after S11)

> **Self-contained.** Repo conventions (branch/commit/PR, gates, stop-and-report) live in
> `AGENTS.md` / `CLAUDE.md`. **Done = the acceptance checks below pass AND the maintainer has been
> handed the tag step.** The PyPI publish is irreversible and outward-facing — **do not push the tag
> yourself; stop and hand off.** Stop and report on any blocker.

## Mission

Finalize 1.3.0: bump the package version, promote the changelog, run the final gates, and confirm
`cti.__version__ == "1.3.0"`. Then **hand the tag/publish to the maintainer** — pushing `v1.3.0`
triggers CI to build all-platform wheels + sdist and publish to PyPI. "Done" for this session = the
version-bump PR is merged to `main`, green, with `cti.__version__` confirmed, and the tag step
clearly handed off.

## Context

- **The dep is already `1.3.0`** (S1). Only `[package] version` lags at `1.2.2` — a deliberate
  staged-release choice so `main` didn't advertise `1.3.0` mid-stream. This batch flips it. That
  **single edit drives both** the wheel version and (via S7's `importlib.metadata` mixed layout)
  `cti.__version__`.
- **Release is tag-triggered.** `.github/workflows/CI.yml`'s `release` job enters on `refs/tags/*`
  (also on `workflow_dispatch`, but its `uv publish` step is itself gated to `refs/tags/*`, so
  publishing only happens on a tag), needs `verify` + the linux/musllinux/windows/macos wheel jobs +
  `sdist`, and publishes via `uv publish` with `UV_PUBLISH_TOKEN` (set from
  `secrets.PYPI_API_TOKEN`). **Tag convention is `vX.Y.Z`** (existing tags:
  `v1.2.2`, `v3.0.5`, …) → this release is **`v1.3.0`**.
- Must already be merged: **S7** (so `__version__` resolves) and **S11** (so the 1.3.0 tag contains
  `docs/DECISIONS-1.3.0.md`).
- Past releases rename `## [Unreleased]` straight to the version (no empty `[Unreleased]` is kept
  afterward — see `## [1.2.2] - 2026-04-04`). Follow that pattern.

## Changes (in order)

1. **`Cargo.toml` — bump `[package] version` `"1.2.2"` → `"1.3.0"`.** Leave the dependency at `1.3.0`.
   Nothing else.
2. **`Cargo.lock`** — regenerated by the next build (this package's own version updates). Stage it.
3. **`CHANGELOG.md` — promote, don't append.** Rename `## [Unreleased]` to `## [1.3.0] - <YYYY-MM-DD>`
   (today's date), keeping the consolidated Fixed/Added/Changed as-is. Do **not** leave a stray empty
   `[Unreleased]` (matches the repo's prior pattern).
4. **Build + confirm the version.** `maturin develop` (rebuilds with the new dist version), then
   `python -c "import centaur_technical_indicators as c; print(c.__version__)"` → **must be
   `1.3.0`**. If it still prints `1.2.2`, the editable dist-info is stale — reinstall and re-check;
   if it persists, stop and report.
5. **Gates:** `python -m pytest` (expect the full suite green — 112+), `cargo fmt --all -- --check`.
6. **PR (`release/1.3.0` → `main`)**, merge.
7. **HAND OFF THE TAG — do not push it.** Tell the maintainer: tag the merged commit `v1.3.0` and
   push it; CI's `release` job builds all-platform wheels + sdist and runs `uv publish` to PyPI.
   This is the irreversible publish — it is the maintainer's action, not this session's.
8. **Final housekeeping (only after publish is confirmed):** remove the untracked scratch
   (`PLAN.md`, `RESUME.md`, `*_BRIEF.md`) — decisions now live in `docs/DECISIONS-1.3.0.md` (S11).
   This is a plain `rm` of never-committed files (not a git change).
